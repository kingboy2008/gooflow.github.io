Data Definition

Global Configuration

These configurations need to be set before initializing a GooFlow instance.

GooFlow.colors Type: Json Object

The default, configurable color of each major element in GooFlow.

PropertyTypeDefaultDescription
mainString#20A0FFThe color of the designer title bar, top divider, toolbar button background.
fontString#15428BThe default text color.
nodeString#C0CCDAThe default normal node background color.
lineString#1D8CE0The color of lines, and the color of the border when node is selected.
lineFontString#777The color of the text (the priority is greater than "the default text color").
markString#ff8800The color of the line or node when it is marked; the color of the line when it is selected.
mixString#B6F700Composite node default background color.
mixFontString#777The default text color under compound nodes.

GooFlow.remarks Type: Json Object

The default, configurable text comment descriptions for the various action buttons in the designer (shown by the native dom's title attribute).
If nothing is set, there will be no text prompts on all buttons of the designer.

PropertyTypeDefaultDescription
headBtnsJson(empty{})Text description of each type of button in the top bar. Format:
key is the type name, value is the type of text description, such as: {"undo":"undo operation",……}
toolBtnsJson(empty{}) Text description of various types of buttons on the toolbar (including node types and pointers, lines, and lane edit switches). Format:
key is the type name, value is the type of text description, such as: {"cursor":"cursor", "direct":"solid line",……}
extendRightString(undefined)The drawing workspace prompts for extending the button to the right.
extendBottomString(undefined)Textual hints for the drawing workspace down extension button.

Sensitive Keywords

Node Definition

The user can customize the new type, or override the css style of the native own type. However, you cannot name the new node Type with the following keywords:

'cursor'、'direct', 'dashed' and 'dashed'

These keywords are represented in the toolbar: cursor button, solid line button and dotted line button.

Toolbar Description

The text description defines the JsonMap structure {'button type name':'description'} to be constructed in a way; in addition to the node type buttons, the description text definitions of several other buttons need to be set with a key through several keywords. as follows:

GooFlow.remarks.toolBtns = {
    cursor:"cursor button's keyword",
    direct:"solid line button's keyword",
    dashed:"dotted line button's keyword",
    group:"lane editing switch button's keyword",
    ……
};

Head Bar Description

The head bar button already comes with the following Type:

"new","open","save","undo","redo","reload","print"

Among them, undois the undo action button, and redois the redo action button; these two buttons have special functions and cannot be custom events.
The other buttons have special event hooks, see the "Event Hooks" chapter.
