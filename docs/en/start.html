Overview

GooFlow is a web-based process designer that is easy to use, easy to use, and beautiful in interface. It is based on Jquery development and inherits the experience of operating experience like PhotoShop. It can describe a process in a specific Json format and present it to users. It provides rich operating functions for users to view or edit instantly, and it can be easily customized to expand into an online IDE by providing rich interfaces. Believe that both developers and end users will love GooFlow.

Quick Installation

Make sure your machine is properly installednode.js and npm.

Firstly checkout the repository from GitHub or Gitee to your local, and you can run the installation command:

$ npm install

Then run the build command:

$ npm run build

This will generate the latest GooFlow archive file in the dist directory. Finally start the project comes with the http-server program, open the native static Web service:

$ npm run start

The browser will automatically open the project home page, from the above instance link can open to browse the current several demonstration examples.

First Instance

Let us begin the first simplest example.

Introduces jquery, GooFlow style files and core library files in demo.html.

<!-- demo.html -->
<head>
    <link rel="stylesheet" type="text/css" href="./dist/GooFlow.min.css"/>
</head>
<body>
    <div id="demo"></div>
    ……
    <script type="text/javascript" src="https://cdn.bootcss.com/jquery/1.12.4/jquery.min.js"></script>
    <script type="text/javascript" src="./dist/GooFlow.min.js"></script>
    <script type="text/javascript" src="./dist/GooFlow.export.min.js"></script>
    <script type="text/javascript" src="./dist/GooFlow.print.min.js"></script>
    <script type="text/javascript" src="./demo.js"></script>
</body>
//// demo.js
    var options = {
        toolBtns:["start round mix","end round","task","node","chat","state","plug","join","fork","complex mix"],
        haveHead:true,
        headLabel:true,
        headBtns:["new","open","save","undo","redo","reload"],// define header's buttons when haveHead=true
        haveTool:true,
        haveGroup:true,
        useOperStack:true
    };
    var demo = GooFlow.init("#demo",options);

Open demo.html and you'll see a designer page for a blank drawing area! Experience the thrill of painting flowcharts!

AMD Instance

Of course, you can also load library files in AMD mode, but the initialization method is the same:

/** require.config.js **/
requirejs.config({
    map: {
		'*': {
			'css': 'https://cdn.bootcss.com/require-css/0.1.10/css.min.js' // https://github.com/guybedford/require-css, RequireJs's plugin
		}
	},
    paths: {
        jquery: 'https://cdn.bootcss.com/jquery/1.12.4/jquery.min',
        GooFlow: './dist/GooFlow.min',
        'GooFlow.export': './dist/GooFlow.export',  // Optional, a extension file of exporting the flowchart into a picture file.
        'GooFlow.print': './dist/GooFlow.print',    // Optional, a extension file of print the flowchart or save flowchart as a PDF file.
    },
    //// ……
    shim: {
        GooFlow: {
            deps:['css!./dist/GooFlow.min.css','jquery']
        }
    },
});
<body>
    <div id="demo"></div>
    ……
    <script data-main="main.js" src="https://cdn.bootcss.com/require.js/2.3.5/require.min.js"></script>
	<script src="../assets/js/require.config.js"></script>
</body>
/** main.js using extension packages **/
require(['jquery', 'GooFlow'], function ( $, GooFlow ) {
    require(['GooFlow.export','GooFlow.print'], function (){
        // 初始化的代码
        var options = { …… };
        var demo = GooFlow.init("#demo",options);
        demo.setNodeRemarks(remark);         //remarks is a title prompt definition for the left toolbar buttons
        demo.setHeadToolsRemarks(headBtns);  //headBtns is a title prompt definition for the header bar's buttons
        demo.loadData(jsondata);             //jsondata is a json data for expressing the detail of flowcharts
        demo.onBtnSaveClick=function(){
            demo.exportDiagram('myFile');//the function of exporting a png file
        }
        demo.onPrintClick=function(){
			demo.print();//the function of printing or save flowchart as a PDF file
        }
    });
});

User Interface

as the picture shows:


Workspace, its width and height is initially the size without the scroll bar. When the mouse slides to the right/bottom side, a half-tone layer appears. Just click on it to manually expand the workspace. The horizontal X-axis direction isfrom left to right, and the vertical Y-axis direction is from top to bottom, which are the same as the CSS definition. When loading a drawn json flowchart data, the width and height of the workspace will also adaptively load the width and height of the flowchart and automatically expand.

The drawing toolbar is on the left of the drawing area, from top to bottom: selection pointer, connection line, start node, end node, task node, automatic node, decision node, status node, extension additional node, branch node, aggregate node Point, composite sub-process node. The last one is not a node, but a switch for node connection editing and grouping lane editing two work states.

The header's bar is the place where the flowchart titles are displayed and where some function buttons are placed. These buttons can be customized by the user in addition to the system.

If you do not set the left toolbar when it is initialized, the editing function is turned off and the component becomes a flowchart viewer/tracker.

Charts's Elements

In combination with the complete screenshot in the last section and the selected element style table above, you can find:

  • Nodes defaults to a light blue rounded rectangle; if a .item_mix class is added, it well be green；if a .item_roundclass is added, it well become a circle.
  • Nodes are renamed, double-click the text, drag, resize, add and delete as default functions, and they can customize right-click events.
  • Lines have two styles which are a solid line and a dotted line.There are three types of line: a straight line, a middle line that can be dragged up and down, and a middle line that can be dragged left and right. Double-click the line to directly edit the text description of the line. Manually change the start or end of the connection.
  • Grouping lanes, the switches in the lower left corner are required to be edited after being pressed. There are four random colors. Click the circle icon in the upper left corner to change the color. Double-click the text to directly edit, or drag and resize as the node.
